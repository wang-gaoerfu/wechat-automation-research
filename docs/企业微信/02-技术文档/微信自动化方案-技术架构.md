## 二、智能助手 (Agent) 设计

### 2.1 Agent 工作流程

```
用户: "给所有VIP客户发送端午节活动通知"

    │
    ▼
┌─────────────────┐
│  自然语言理解   │
│  (LLM解析意图) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    意图识别     │
│  intent: broadcast │
│  target: VIP客户  │
│  content: 端午节活动 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    任务拆解     │
│                 │
│  1. 查询VIP客户 │
│  2. 过滤活跃客户│
│  3. 分批发送   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   工具调用      │
│                 │
│  customer_mcp   │
│  └ list_customers │
│  └ filter_by_tag  │
│                 │
│  wechat_mcp     │
│  └ send_message  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    结果反馈     │
│                 │
│ "已发送给128位  │
│  VIP客户，发送  │
│  成功率100%"   │
└─────────────────┘
```

### 2.2 支持的指令类型

| 指令类型 | 示例 | 调用的MCP工具 |
|---------|------|-------------|
| 客户查询 | "查询本周新增客户" | `customers.list`, `customers.filter` |
| 消息发送 | "给华东区客户发通知" | `message.send`, `customers.filter` |
| 群发任务 | "创建每周一早报推送" | `scheduler.create_task` |
| 知识库 | "添加FAQ到知识库" | `knowledge.add_document` |
| 数据统计 | "本月发送了多少条消息" | `stats.get_monthly_report` |
| 配置修改 | "开启新客户自动标签" | `config.update` |

### 2.3 Prompt 设计

```python
SYSTEM_PROMPT = """
你是一个微信运营助手，帮助用户管理企业微信客户。

你有以下能力：
1. 查询客户列表、筛选客户
2. 发送消息给客户或群
3. 创建定时任务
4. 管理知识库
5. 查看运营统计数据

执行规则：
- 发送消息前检查频率限制，避免触发风控
- 群发消息需要用户确认后再执行
- 涉及敏感操作需要二次确认

回复格式：
- 操作成功：简洁报告结果
- 操作失败：说明原因并建议解决方案
- 需要确认：列出操作详情，等待用户确认
"""
```

## 四、双方案技术对比

| 维度 | 企业微信方案 | 个人微信方案 |
|------|-------------|-------------|
| **MCP封装** | `WeChatWorkMCP` | `WeChatFerryMCP` |
| **消息接收** | Webhook回调 | Hook拦截 |
| **消息发送** | 官方API | RPC调用 |
| **频率控制** | API限制 + 本地计数 | AntiBanEngine |
| **内容审核** | 不需要 | ContentDiversifier |
| **LLM对接** | 支持 | 支持 |

## 五、Agent 业务场景示例

### 场景1：智能客户跟进

```
用户: "给3天没互动的VIP客户发一条关怀消息"

Agent处理:
1. 调用 customer_mcp.list_customers(tag="VIP")
2. 过滤出最后互动时间 < 3天的客户
3. 调用 message_mcp.send_broadcast() 发送关怀消息
4. 返回发送结果统计
```

### 场景2：智能知识问答

```
用户: "有客户问如何收费，在知识库查一下"

Agent处理:
1. 调用 knowledge_mcp.search(query="收费", top_k=3)
2. 整理搜索结果
3. 返回："根据知识库，收费标准是..."
```

### 场景3：定时内容推送

```
用户: "每周五下午6点给所有客户发送周末祝福"

Agent处理:
1. 解析 cron 表达式: "0 18 * * 5"
2. 调用 scheduler_mcp.create_task()
3. 返回: "已创建每周五18:00的定时任务"
```
