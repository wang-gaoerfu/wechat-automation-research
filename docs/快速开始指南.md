## 一、环境准备

### 1.1 基础要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10/11 或 Linux |

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.9 或更高版本（推荐 3.11）

验证安装：
```bash

pip --version
```

## 三、大模型配置（SenseNova）

### 3.1 获取 API Key

1. 访问 https://token.sensenova.cn/
2. 注册账号并完成手机号验证
3. 在控制台 → API Keys 创建 API Key（sk- 开头）

### 3.2 支持的模型

| 模型 ID | 说明 | 调用限制 |
|--------|------|---------|
| sensenova-6.7-flash-lite | 轻量多模态模型 | 每5小时1500次 |
| deepseek-v4-flash | DeepSeek 高性能模型 | 每5小时150次 |

### 3.3 配置示例

在 `config.yaml` 中配置：

```yaml
# LLM 大模型配置
llm:
  provider: "sensenova"                     # 使用 SenseNova
  api_key: "sk-your-api-key"                # 你的 API Key
  base_url: "https://token.sensenova.cn/v1" # 不填则使用默认地址
  model: "sensenova-6.7-flash-lite"         # 模型名称
  temperature: 0.7
  max_tokens: 2000
```

## 四、知识库配置

### 4.1 启用知识库 RAG

```yaml
# RAG 知识库配置
rag:
  enabled: true
  embedding_provider: "sensenova"            # 使用 SenseNova 做 Embedding
  embedding_api_key: "sk-your-api-key"       # 与 LLM API Key 相同
  embedding_base_url: "https://token.sensennova.cn/v1"
  collection_name: "knowledge_base"          # 默认知识库名称
  persist_directory: "./data/chromadb"       # 向量数据持久化目录
  similarity_threshold: 0.5                  # 相似度阈值
```

### 4.2 多知识库场景

系统支持多个独立的知识库，适用于不同客户/场景：

```yaml
# 可以在配置中指定默认知识库名
rag:
  collection_name: "客户A_知识库"
```

## 五、企业微信方案

### 5.1 准备企业微信账号

#### 步骤1：注册企业微信

1. 访问 https://work.weixin.qq.com/
2. 点击 "注册"
3. 选择 "企业" 或 "个人工商户"
4. 填写企业信息完成注册

#### 步骤2：创建应用

1. 登录企业微信管理后台：https://work.weixin.qq.com/wework_admin/
2. 进入 "应用管理"
3. 点击 "创建应用"
4. 填写应用信息并创建

#### 步骤3：获取应用凭证

在应用详情页获取：
- `AgentId`（应用ID）
- `Secret`（应用密钥）
- `CorpID`（在"我的企业"→"企业信息"中获取）

### 5.2 安装与配置

**进入企业微信项目目录并创建虚拟环境：**
```bash
cd D:\my_project\ai_projects\wechat-automation-research\enterprise-wechat
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**配置：**
```bash
cp config\config.yaml.example config\config.yaml
```

编辑 `config\config.yaml`：

```yaml
wechat:
  corp_id: "ww你的企业ID"
  corp_secret: "你的应用Secret"
  agent_id: "你的应用AgentId"
  callback_token: "你设置的Token"
  callback_aes_key: "你设置的EncodingAESKey"

# LLM 大模型配置（SenseNova）
llm:
  provider: "sensenova"
  api_key: "sk-your-api-key"
  base_url: "https://token.sensenova.cn/v1"
  model: "sensenova-6.7-flash-lite"

# RAG 知识库配置
rag:
  enabled: true
  embedding_provider: "sensenova"
  embedding_api_key: "sk-your-api-key"
  embedding_base_url: "https://token.sensenova.cn/v1"
  collection_name: "enterprise_kb"
  persist_directory: "./data/chromadb"

# Agent 智能助手配置
agent:
  enabled: true
  use_rag: true

app:
  host: "0.0.0.0"
  port: 8000
  debug: false
```

### 5.3 启动服务

```bash
python run.py
```

访问：http://localhost:8000/docs

## 七、知识库管理 API

### 7.1 上传文档到知识库

```bash
# 上传文档（企业微信端口 8000，个人微信端口 8001）
curl -X POST http://localhost:8000/api/v1/kb/upload \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "product_info.txt",
    "content": "产品名称：智能助手\n功能：自动回复、定时群发...",
    "collection": "产品知识库"
  }'
```

### 7.2 上传文件

```bash
curl -X POST http://localhost:8000/api/v1/kb/upload/file \
  -F "file=@./docs/product.md" \
  -F "collection=产品知识库"
```

### 7.3 添加监控路径（自动同步）

```bash
# 监控指定目录，文件变化自动更新知识库
curl -X POST http://localhost:8000/api/v1/kb/watch \
  -H "Content-Type: application/json" \
  -d '{
    "path": "D:/my_project/knowledge",
    "collection": "产品知识库",
    "file_types": [".txt", ".md", ".pdf"],
    "recursive": true
  }'
```

### 7.4 启动文件监控

```bash
curl -X POST http://localhost:8000/api/v1/kb/watch/start \
  -d "interval=60"
```

### 7.5 列出所有知识库

```bash
curl http://localhost:8000/api/v1/kb/collections
```

### 7.6 跨知识库搜索

```bash
curl -X POST http://localhost:8000/api/v1/kb/search/multi \
  -H "Content-Type: application/json" \
  -d '{
    "query": "产品价格",
    "collections": ["产品知识库", "FAQ知识库"],
    "top_k": 5
  }'
```

## 九、常见问题

### Q1：提示 "LLM 服务未初始化"？

**解决**：检查 `config.yaml` 中 `llm.api_key` 是否正确配置

### Q2：知识库搜索没有结果？

**解决**：
1. 确认已上传文档到知识库
2. 检查 Embedding API Key 是否正确
3. 尝试降低 `similarity_threshold`（如设为 0.3）

### Q3：SenseNova API 调用失败？

**解决**：
1. 确认 API Key 有效且额度充足
2. 检查 Base URL 是否为 `https://token.sensenova.cn/v1`
3. 确认模型 ID 正确（如 `sensenova-6.7-flash-lite`）

### Q4：文件监控不生效？

**解决**：
1. 调用 `POST /api/v1/kb/watch/start` 启动监控
2. 检查监控路径是否存在
3. 确认文件格式受支持（.txt, .md, .csv, .json）

## 十、API 接口一览

### 企业微信 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /health` | GET | 健康检查（含LLM/RAG状态） |
| `POST /api/v1/agent/chat` | POST | Agent 智能对话 |
| `POST /api/v1/kb/upload` | POST | 上传文档 |

| `GET /api/v1/kb/collections` | GET | 列出知识库 |
| `POST /api/v1/kb/search/multi` | POST | 跨知识库搜索 |
| `POST /api/v1/kb/watch` | POST | 添加监控路径 |

### 个人微信 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /` | GET | 服务状态（含LLM/RAG状态） |
| `GET /ready` | GET | 就绪状态 |
| `POST /api/v1/kb/upload` | POST | 上传文档 |
| `POST /api/v1/kb/upload/file` | POST | 上传文件 |

## 十一、安全建议

1. **API Key 安全**：不要把真实的 API Key 提交到 GitHub
2. **配置文件**：使用 `.gitignore` 忽略 `config.yaml`
3. **生产环境**：使用环境变量而非配置文件存储敏感信息
4. **HTTPS**：生产环境务必启用 HTTPS
